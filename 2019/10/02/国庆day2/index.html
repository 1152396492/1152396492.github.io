质数求和

题意

输入两个数 L、R，输出所有 L 到 R 之间(包括 L、R)的质数的和。
对于 30%的数据， 1<=L<=R<=1000
对于 100%的数据，1<=L<=R<=10^9，R-L<=1000

思路

这题你要是打线性筛就完蛋了
观察数据 R - L <= 1000
为什么要给这个条件？
如果线性筛时间是够，空间正好爆炸（耶）

补充线性筛：

原理：

每个合数只被它的最小质因子筛一次
为什么 if( i % prime[j] == 0 ) break; 要放在 notp[i * prime[j]] = 1; 后面呢？难道不能放在它前面吗？
在触发 if( i % prime[j] == 0 ) break; 意味着prime[j]是i最先遇到的质因子，并且我们知道某一个数的某个因子一定小于等于它本身，所以显而易见，我们的素数序列包括到i（无论i是否是质数）的所有质数（如果i是质数，那么包括i）。我们之前也说过这个质数序列是单调递增的，那么prime[j]就是 i 的最小质因子！

树

题意

给出一棵 n 个结点的树，结点编号 0~(n-1)，根结点编号为 0，每一个结点上 有一个正整数权值，你的程序必须维护 4 种操作:
F u v:找出从 u 到 v 的路径上所有点权值的最大公约数。(包括 u 和 v) C u v d:将从 u 到 v 的路径上所有点权值加上 d。(包括 u 和 v)
G u:找出以 u 为根的子树中所有点权值的最大公约数。(包括 u)
D u d:将以 u 为根的子树中所有点权值加上 d。(包括 u)

第一行包含一个正整数 n，表示这棵树的结点个数。接着 n-1 行，每行包含 2 个整数 a 和 b，表示 a 和 b 直接相连的边。
在这之后的一行包含 n 个空格隔开的正整数，依次表示每个结点的初始权 值。然后一行一个正整数 Q，表示操作的个数。
最后 Q 行每行一个询问，格式已由问题描述给出。
1<=n<=50000，1<=Q<=10^5 初始权值在 1 和 10000 之间。 修改操作中的 d 在 0 和 10000 之间。 有 50%的数据除了 n、Q 外几乎随机
