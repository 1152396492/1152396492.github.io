逃离(escape)

题意

小熊维尼醒来时，发现他正处于一个迷宫中。迷宫是一个 n * n 的棋盘，迷宫的左上角 为(1,1)，右下角为(n,n)。他正处于迷宫(1,1)的位置，要想逃出迷宫必须要到达(n,n)。每个时 刻，小熊维尼都可以进行移动，但是，他只能向下或向右移动，即只能从(i,j)移动到(i+1,j) 或(i,j+1)，同时，也不能够移动出棋盘。跳跳虎发现小熊维尼被困在迷宫中，也来捣乱，他 站在棋盘的右上角，准备阻止小熊维尼走出迷宫，因此，小熊维尼不得不远离它的控制范 围，必须时刻保证自己处于棋盘的左下半区(即(x,y),x>=y)。同时，小熊维尼会记录他行走 的路径，每向下走一步，他将在纸上写下一个’D’;每向右走一步，他将在纸上写下一 个’R’。终于，小熊维尼走出了这个迷宫。然而他并不满足于走出迷宫，他现在已经开始盘 算着如果下次进入迷宫应该怎么走啦。准确的说，小熊维尼想要每次走出迷宫的路径都比 上次走出的路径的字典序更大。具体在现在，小熊维尼想知道，如果想要得到的字符串刚 好比这次的大，下次他应该如何走出迷宫?
注:路径序列 y 的字典序刚好比 x 大，即满足 x < y 且不存在合法路径序列 z 使得 x < z < y
对于 30%的数据，n ≤ 16
对于 60%的数据，n ≤ 2000 对于 100%的数据，n ≤ 1000000

思路

右移能移动的D，因为R变成D不影响，所以只考虑D ，将能移动的D以编号从大到小排序，取最右边的，如果能变化就变，不能就退到下一个。
什么情况不能变：
1、变了后被跳跳虎控制
2、没有用来转换的R
如果转换了D要把它后面所有的D全部移到它后一个去，保证最大

队列(queue)

题意

队列是一种常见的数据结构，具有先进先出的特性，有一天，小熊维尼也有了这样一 个队列。接下来，跳跳虎给了小熊维尼一些数字，以及一些操作。
操作有两种:
1 x:向队尾中加入一个数字 x。
2:弹出队首元素，若队列中没有元素，则保持原状。 每次操作之后，跳跳虎都要小熊维尼回答队列中的所有数字的乘积是多少。小熊维尼当
然不会啦，所以……你来帮帮他咯。 由于数据较大，将用以下函数代替读入。
unsigned myrand(){
unsigned s3=s1,s4=s2; s1=s4;
s3^=s3<<23; s2=s3^s4^(s3>>17)^(s4>>26); return s2+s4;
} input:
第一行，4 个整数 n,s1,s2,MOD。
之后的输入由 myrand()函数代替。
接下来 n 行，描述 n 次操作，先输入一个数字 p(p=2-(myrand()&1))，根据 p 对应的
操作输入接下来的数据(若 p=1，则输入 x=myrand()^lastans)。 其中 lastans 为你上一次输出的值，初始时为 0。
output:
在每一次操作之后，输出队列中所有元素的乘积。 由于乘积可能很大，输出乘积对 MOD 取模之后的值。 由于输出太多，因此你只需要在程序最后输出即可，假设每次操作之后的答案为
ans ，你只需要输出sum = ans2^ans2^ … ^ans2
对于 30%的数据，n ≤ 5000
对于 50%的数据，n ≤ 300000
对于另 20%的数据，MOD 是小于106的质数
对于 100%的数据，1 ≤ n ≤ 10000000,3 ≤ MOD ≤ 109,0 ≤ 𝑠𝑠1,𝑠𝑠2 < 232

【样例输入】 5 1 1 100 【样例输出】
4672 【样例解释】
实际输入为 5 1 1 100
1 8388673
2 1 536879241
2 2
实际输出为 73
0 41
0 0
最终结果4672 = (73^2 ^ 41^2)

思路

30分：
开一个队列模拟，每次暴力求和
50分：
线段树求积
另外20分：
预处理0 ～ mod-1所有的乘法逆元，直接维护队列的积
加入的话
sum = sum * x % mod ;
弹出的话:
sum = sum * inv ( x ) % mod ;
100point:

设一个分界线把队列分成左右两段，左半段记录所有位置到分界
线的后缀信息，即每个后缀的乘积，
右半段只维护所有数字的乘
积，每次询问就将左右两边乘起来。入队直接在右半段添加，出
队操作时，如果左半段非空，则直接弹掉队首，如果已经空了，
则暴力地将右半段中所有的数字拿到左边，并重新计算出所有后,
缀信息即可，这样每个数字在维护的过程中最多只会被做2次乘
法。
●时间复杂度0(n)

BZOJ4709柠檬

给定一个长度为n的序列a。你需要将它分为任意多
段。
对于每一段，你需要指定一个数X，如果这一段内X
的个数为k，那么价值为xk^2.

最大化价值总和
n≤10^6

单调队列优化（不会的

如果一个区间左端点的数字和这个区间选取的x不同,那么我们把这个左端点拿出来单独成为一个区间肯定可以得到更优的解.也就是说,最优解中每个区间的左右端点的数字都等于这个区间选取的x
f[i]表示前i个数的最大价值和。
f[i]=max{f[j]+ai2}，其中a[j+1]=a[i]。
对每个a[i]分别维护决策，不难发现决策是单调的。
时间复杂度O(nlogn)

斜率优化

有些DP方程可以转化成DP[i]=max{f[j]}+x[i]的形式，其中f[j]中保存了只与j相关的量。这样的DP方程我们可以用单调队列进行优化，从而使得O(n^2)的复杂度降到O(n)
但像这样的方程：dp[i]=max{dp[j]+(x[i]+x[j]) * (x[i]-x[j])}。如果把右边的乘法化开的话，会得到x[i] * x[j]的项。这就没办法使得f[j]里只存在于j相关的量了。于是上面的单调队列优化方法就不好使了。
这里学习一种新的优化方法，叫做斜率优化。

HDU 3507 print artical

题目大意：输出N个数字a[N]，输出的时候可以连续的输出，每连续输出一串，它的费用是 “这串数字和的平方加上一个常数M”。n<=500000
我们设dp[i]表示输出到i的时候最少的花费，sum[i]表示从a[1]到a[i]的数字和。于是方程就是：
dp[i]=dp[j]+M+(sum[i]-sum[j])^2；
很显然这个是一个1D1D的转移。题目的数字有500000个，不用试了，二维铁定超时了。那我们就来试试斜率优化吧，看看是如何做到从O(n^2)复杂度降到O(n)的。

我们假设k < j < i。如果在j的时候决策要比在k的时候决策好，那么也是就是dp[j]+M+(sum[i]-sum[j])^2 < dp[k]+M+(sum[i]-sum[k])^2。(因为是最小花费嘛，所以优就是小于)
两边移项一下，得到：(dp[j]+sum[j]^2-(dp[k]+sum[k]^2))/(2 * (sum[j]-sum[k])) < sum[i]。我们把dp[j]+sum[j]^2看做是yj，把2 * sum[j]看成是xj。
那么不就是yj-yk/xj-xk < sum[i]么？ 左边是不是斜率的表示？
那么(yj-yk)/(xj-xk) < sum[i]说明了什么呢？
说明g[j,k]=(yj-yk)/(xj-xk ) < sum[i]代表这j的决策比k的决策要更优。
关键的来了：若k < j < i且g[i,j] < g[j,k]，则j点永远不可能成为最优解，可以直接将它踢出我们的最优解集。为什么呢？
分三种情况讨论：
设当前点为a
1.如果g[i,j]与g[j,k]均小于sum[a]，则i比j优，j比k优
2.如果g[i][j]与g[j,k]均大于sum[a],则k比j优，j比i优。
3.如果g[i][j] < sum[a]且g[i][j] > sum[a]，则i比j优，k比j优。
不论如何，j都无法成为最佳决策点，所以可以排除j。
于是，所有的决策点满足一个下凸包性质。
接下来看看如何找最优解。
设k < j < i。
由于我们排除了g[i,j] < g[j,k]的情况，所以整个有效点集呈现一种下凸性质，即g[i,j] > g[j,k]。
这样，从左到右，斜率之间就是单调递增的了。当我们的最优解取得在j点的时候，那么k点不可能再取得比j点更优的解了，于是k点也可以排除。换句话说，j点之前的点全部不可能再比j点更优了，可以全部从解集中排除。
于是对于这题我们对于斜率优化做法可以总结如下：
1，用一个单调队列来维护解集。
2，假设队列中从头到尾已经有元素a b c。那么当d要入队的时候，我们维护队列的下凸性质，即如果g[d,c] < g[c,b]，那么就将c点删除。直到找到g[d,x]>=g[x,y]为止，并将d点加入在该位置中。
3，找最佳决策点时，设当前求解状态为i，从队头开始，如果已有元素a b c，当i点要求解时，如果g[b,a] < sum[i]，那么说明b点比a点更优，a点可以排除，于是a出队，直到第一次遇到g[j,j-1] > sum[i]，此时j-1即为最佳决策点。
没有三个点i， j，k。丼中xi>xj>xk。如何判断三点是上凸逐是下凸?
用向量的叉积运算即可c
b没有向量p1(x1,y1)， p2(x2,y2)。 如何判断p1与p2的位置关系?
若p1xp2>0，則p2在p1 的逆肘針方向;
ゝ若p1xp2<0， 則p2在p1 的順肘針方向;
ゝ若p1xp2=O， 則p1与p2方向重合。
ゝp1xp2=(x1 * y2 -x2 * y1)
ゝ所以我们 可以令向量p1=j-k，p2=i-j. 再用叉和印可剣断i，j，k是上凸还是下凸。
ゝ另外注意:比較斜率避免用除法。

TopCoder SRM 666 Div2 Problem 999 - WalkOverATreeDiv2

题意：
给一个具有n个节点的树，每个节点上有一个值v，现在从节点1开始，走L步，问L步中经过的v值之和最大为多少。若一个点被重复走过，这个权值只计算一次。
范围：
1 < n <=50，0<=L<=100，0<=v<=100。

树形DP解决，状态分析：
dp[u][1][step]：表示从节点u开始，走step步，此时回到u点，经过的点的最大权值之和。
dp[u][0][step]：表示从节点u开始，走step步，此时没有回到u点，经过的点的最大权值之和。
状态转移：
走step步回到u点，只有一种转移方法：child表示当前的子树，我们都从当前子树考虑，若能转移，当前子树也必须回到child点。所以转移方程为：
dp[u][1][step]=max(dp[u][1][step]，dp[u][1][i]+dp[child][1][step-i-2])：dp[u][1][i]从前面的几个孩子求得的值，dp[child][1][step-i-2]当前孩子必须回到child点，-2表示u->child的边，来回的步数。
走step步没有回到u点，有两种转移方法：也是从当前子树考虑。若能转移，可以有两种情况：
①当前子树回到child点，前面考虑的子树没有回到u点；
dp[u][0][step]=max(dp[u][0][step]，dp[u][0][i]+dp[child][1][step-i-2])
②当前子树没回到child点，前面考虑的子树回到了u点。
dp[u][0][step]=max(dp[u][0][step]，dp[u][1][i]+dp[child][0][step-i-1])
