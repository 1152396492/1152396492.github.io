Tarjan算法的原理及使用方式

发表于 2019-06-09 | 分类于 代码

一.算法简介

Tarjan 算法一种由Robert Tarjan提出的求解有向图强连通分量的算法，它能做到线性时间的复杂度。

我们定义：

  • 强连通：有向图中，两个顶点至少存在一条路径。
  • 强连通图：没两个顶点都强连通都有向图。
  • 强连通分量(strongly connected components)：有向图的极大强连通子图。

二、什么是强连通分量？

例如在下图中：
一个点，我们视为一个强连通分量，例如4这个点是强连通分量，2，3，4这三个点是强连通分量。但由于4可以添加2，3进去，所以4不是极大的强连通子图，2、3、4是极大的强连通子图，因为它们没办法扩展。所以在下图中：1是一个强连通分量，2、3、4是一个强连通分量。

强连通意味着两两可达，当我们遇到有环的问题，可将这些环缩成一个点，也就是把一个有环图的问题转化成一个树或森林的问题，求强连通分量后，如果在这些被缩成一个点的点中的数据，和有用就求和，最大值有用就求最大值，从而解决实际问题。

那么我们缩图要缩成什么样子呢？下图：


1、2、3、4是两两可达的，6，7，8是两两可达的，5自身可看作能到达自己。于是我们可以将1、2、3、4缩成一个假设为scc1的点，6、7、8缩成一个scc2 的点，5单独为一个scc3的点。不难发现缩成点后，每一个点内的边都不存在了，而且，每一个点的出边是强连通分量的出边，每个点的入边是强连通分量的入边。

三、那么具体要怎么缩点呢？

有两种方式：1、Kosaraju算法和tarjan算法。虽然Kosaraju算法比Tarjan要好理解的多，但实现时与实际应用范围也更小。

这里我们讲Tarjan。
使用搜索从任意节点遍历有向图时，可以得到搜索树。

每一条边和搜索树的关系可以分为下面四种：

  1. 树枝边：搜索树上的边，即未访问过节点的边
  2. 前向边：指向搜索树中子树中节点的边。
  3. 后向边：指向搜索树中父亲（也就是从根节点到这条边的路径中，在它之前被遍历的点）的边。
  4. 横叉边：其他边，即指向搜索树中非子树的边。

下面考虑如何判断这四种边。

我们来看下面这张图：
红色标记的为搜索树，我们走到了8，可以回溯，也就是返回到4，再搜索9
记录搜索的顺序用dfn来表示，例如dfn[1]就代表1节点的搜索的顺序。

我们来定义4种边的类型。

A边终点未访问过，为树枝边。
B边终点已经被访问过，且dfn[v(目标点)]>dfn[u(起始点)]。说明在子树中，所以为前向边。C边终点已被访问过且不在子树中，终点在栈中则为后向边。
D边终点已被访问过且不在子树中且已经出栈，为横叉边。
其中，无向图中不存在前向边和横叉边。

在一个有向有环图上搜索时，考虑每一个强连通分量高度最低（距离最开始搜索的点最近）的那个点。这些点将搜索树分割成了许多个子树，每个子树中的点组成了一个强连通分量。分割的方法是在搜索的同时另外维护一个栈存放节点，离开分割点时把分割点向下的部分全部取出来就是一个强连通分量。

Tarjan的流程：先搜索1，3，4，6，7，8当再次搜索至6时，栈中的元素，从下到上依次为1，3，4，6，7，8。6已经被找过了，所以6是后向边，所以弹出元素8，7，6，把它们缩成一个点；栈中元素从下往上为1，3，4.然后从4搜索至1，但此时1还没有搜索完它所有的节点，所以返回，再从1搜索2，把2加入栈中，再搜索5，搜索5后没有多余节点，弹出5为一个节点。再把1，2，3，4从栈中弹出，这就是Tarjan的算法流程。

那么我们具体怎么实现呢？

我们维护一个数组low，low[u]代表点u所能到达的深度最小的点的dfs（搜索）序编号；初始化low[u]为u节点的搜索的顺序编号dfn[u]。
对于一条边（u，v）（从u出发，v为终点的一条边）

—若为树枝边，因为等到递归再次通过v搜索至其它的点的时候，可能v可以搜索至其它深度更小的点，只要有树枝边，那么必然它的low值被改变的更小，因为low[u]是要求出最小的深度的点，所以用low更新，比用dfn搜索序更新所带来的方案要更优些。则用low[v]更新。
—若为后向边，则用dfn[v]更新。因为low[u]保存的是u节点能够到达的最小深度的点。
—若为前向边，因为指向的点的信息已经通过其他的枝边传递过来，所以无需更新。
—若为横叉边，假设我们出发的这个点是在目标点的强连通分量中，那么我们的出发点就会早已被目标点中的强连通分量中的点更新，所以必然指向的是另一个强连通分量，无需更新。
当我们搜索完成后，low[u] = dfn[u]时，说明u这个点已经完全不能往上走了，所以它就是一个分割点。

搜索时任选一个结点开始搜索，若搜索结束后仍有未访问的点，则再从中任选一个点进行。搜索中对已经访问的点不再访问。
